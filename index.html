Learn Git Command

 What is Git...?

> Git is a version control system, it helps us to track the changes made in our code/projects.
> It is the most popular version control system in software development field.
> It is free and open source software.
> It is fast and scalable, can be used in large project.
> It is use to track the history of code and help to collaborate.

 What is GitHub...?

> Github is a website used by developers to store and manage their codes/projects with the help of Git.
> On GitHub, projects are stored in folder structure that is known as "repository" in sort called "repo".

 Clone Commands :

> Clone: Cloning a repository on our loacal machine.
> CD(Cange Directory): cd fileName+tab
> Out from Directory: cd ..
> Check Version: git -v or git --version
> Clear Terminal: clear
> ls(lists File): ls
> Hidden Files: ls -a
> Clone Link: git clone <-link->

 Status Commands :

> Status: Displays the state of the code.
> Process: M/U => add => Staged => Commit => Unmodified
> Check status: git status
> M: Modified
> U: Untracked File(New File)
> S: Staged(File ready to be committed)
> : Unmodified

 Add and Commit :

> Add: Add new or changed file in working directory to the Git staging area.
> Add File: git add fileName / git add.
> Commit: Record of changes(Final).
> Commit with Message: git commit -m "Meaning Fll Message"
> Check history of Commits: git log

 Push Command :

> Push: Upload local repo content to remote repo
> Push File: git push origin main
> Shorten Push: git push -u origin main => git push

 Init Command :

> Init: Used to create a new git repo
> Make new directory: mkdir newFolder
> New Git: git init
> Git Remote add to origin: git remote add origin <-link->
> Verify Remote: git remote -v

 Branch Command :

> Multi developers works on single project.
> Main branch is the default branch.
> Check Branch: git branch
> Rename Branch: git branch -M main
> To navigate other branch: git checkout <-branch name->
> Create new branch: git checkout -b <-new branch name->
> Delete branch: git branch -d <-branch name->

 Merging Code :

> Compare branches/files: git diff <-branch name->
> Merge 2 branches: git merge <-branch name->
> By Pull Request(PR), we can also merge in GitHub.
> Merge Conflicts: An event that takes place when Git is unable to automatically resolve the differences in code between two commits.

 Pull Command :

> Pull: Used to fetch and download content from a remote repo and immediately update the local repo to match that content.
> Pull File: git pull origin main

 Undoing Changes :

> Case 1: Staged Changes(single) : git reset <-file name->
> Case 2: Staged Changes(many) : git reset
> Case 3: Commited Changes(single) : git reset HEAD~1
> Case 4: Commited Changes(many) : git reset <-commit hash-> > Case 4: Commited Changes(come to original status) : git reset --hard <-commit hash->

 Conclusion > Flow: GitHub repo => clone => changes => add => commit => push
